Lua 源码学习

2022-06-15 0 评论

Lua

Lua主线程的状态：global_State

里面包含了一系列底层函数，包括内存释放函数，内存使用量计数，GC状态，GC托管堆，以及一些具有唯一性的数值。

每一个正在运行的lua线程具备一个lua_State块。

里面包含了：当前正在调用的函数信息，代码栈的指针，异常跳转点，

数据

lua的数据由一个union保存：

TValue组成了一个基础数据，value_是其中保存的数据，tt_是该数据的类型

string

lua 的string的采样的是slab算法。所有的字符值都存储在global_state里面，每一个用户使用的字符仅仅存储一个对应的hash值。

Proto

函数原型里面包括了一个函数的字节码，upvalues，常量表，栈信息。。。。

upvalue保存了目标的数值。

闭包

lua的闭包有以上两种，C函数的闭包，和lua函数闭包。

创建lua闭包的过程：

首先初始化一个空closure，然后将其压入栈，然后寻找对应upval的值并保存下来。寻找对应数值的过程使用的是地址。lua在编译器将变量替换为地址，如果当前已经在一个函数内部，那么内层函数只需要从外部嵌套的函数的upvalue复制出来自己需要的即可，否则luaF_findupval函数从lua_state里面的openupval里面一一对比地址寻找目标变量。如果没找到的话，那么就新建一个并将其链到openupval里面。接下来就可以复用此upvalue.

创建c函数闭包就相对较为简单，其实就是将需要的变量从栈中取出存入upvalue。

CallInfo

callinfo就是一个函数调用的信息。本身是一个双向链表，存储了当前函数的地址，命令。

寄存器

ra就是一个寄存器，i本身就是一条指令，宏RA是在计算当前寄存器的偏移地址，对于A类寄存器，其位置就在指令i的后几bit位。base本身是当前函数的地址起始位，base+偏移地址就是对应的数据的地址。ra就是指向目标数据的指针。

Op表示操作码占7bit

A,B,C表示寄存器的偏移地址

sBx表示指令跳转偏移地址

Bx常量表的偏移地址

lua的执行是函数 luaV_execute，它会首先创建一个closure和StackValue，然后死循环一条条的读取指令并解析。

在每一次执行一个指令时都会生命一个寄存器指针。

lua在调用一个函数对应CALL指令：

如果是一个C函数，程序将直接调用此函数。

如果是一个lua函数，程序将goto startfunc回到解析函数开头，相当于递归了，但是有趣的是，startfunc标记点在声明closure，stackvalue之后，这相当于对递归的优化了。这正好解释了闭包的传递性。

垃圾回收

内存申请：luaC_newobj函数向操作系统申请一块指定大小的内存，然后使用GCObject标记这块内存，最后将其挂载到globa_State的内存链条上。

对于LuaM_newObject每次申请内存时会将内存大小加到 g->GCdebt.

当有大块内存申请时将触发一次垃圾回收检测 checkGC，

lua会去检测当前global_State的GCdebt如果大于零则开始一次回收

首先会先保存当初的指令地址，然后执行垃圾回收过程

所有的垃圾回收对象都具备一个颜色标记位：

  • 白色代表该内存尚未被标记
  • 灰色代表该内存已被标记，但是其引用可能尚未被标记
  • 黑色代表该内存及其引用已被标记

在一次垃圾回收开始时所有的对象都会被标记为白色

lua的垃圾回收包含分代回收(generational)和增量回收(incremental)

分代回收

通过注释 我们可以很清晰的理解分代的算法。

一次普通的垃圾回收被称为minor collection，当内存使用量比上次major collection结束时还大的话，就会进行一次major collection 同时会检测是否回收了足够的内存，至少也得是上次major之后增量的一半都得被回收掉，如果回收的空间不够大，则标记此次回收为bad collection那么下次回收的时候就是一次全量回收。

minor collection

  • 首先标记所有old1类型以及待回收的对象，按照三色标记法标记。

  • 回收掉被标记为白色的对象，对于存活的对象，将会为其标记世代：

​ 对于G_NEW对象的颜色会被标记为白色。并且进入G_SURVIVAL世代。其他世代的对象会保留其颜色然后进入下一世代。

​ 此过程对每一个世代的对象都会单独执行一次。

​ 世代可以分为：

​ |–新创建的对象—|——上一次垃圾回收存活—-|——old1——–|

​ minor collection就只回收前两部分。但是对象被分为自带回收器的对象，以及不带回收器的对象，将会分别执行

  • 收尾阶段，缩减字符串表，重新计算非垃圾内存的评估值。

major collection

触发一次major的条件是：gettotalbytes(g) > majorbase + majorinc

也就是现在的字节数已经大于非垃圾内存的预估值+增量的预估值。

增量的预估值：(majorbase / 100) * getgcparam(g->genmajormul)

一次major collection其实就是一次全量的增量回收

首先会将所有的对象都设置为白色。

然后进行一次全量的增量回收。

增量回收

Lua的增量回收分成了多个步骤完成。主要由函数 static lu_mem singlestep (lua_State *L) 实现。

  • GCPause：标记根节点对象

从当前线程，注册表，元表，final对象，开始标记。

  • GCSpropagate：将一个灰色对象转换为黑色对象
  • GCSenteratomic：这一步是标记阶段的收尾，并且开始按标记回收内存
  • GCSswpallgc，GCSswpfinobj，GCSswptobefnz：就是对不同的内存进行回收
  • GCSswpend：回收字符串

就像增量回收，是分步骤完成的，就会出现一个问题，标记完成之后继续执行程序，此时产生了新的对象，且指向已被标记为黑色的对象，之后的标记时因为不会再扫描黑色对象，那么新对象就会被视为垃圾。因此Lua再生成新对象时会产生ObjectBarrier

**ObjectBarrier **: 创建一个对象时会检测该对象与其对其他变量的引用关系，如果对方是old1世代的变量，那么新变量将会被置于old0.
